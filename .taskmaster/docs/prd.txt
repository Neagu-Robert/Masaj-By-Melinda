Masaj by Melinda – Product Requirements Document (PRD)

---

### 1. Project Overview

Masaj by Melinda is a web application for a professional wellness and massage services business. The platform allows users to explore available treatments, view details, and book appointments online. The site is designed for both desktop and mobile, with a modern, relaxing visual style.

---

### 2. Core Features & Requirements

#### A. Service Presentation

- Display a catalog of massage services and device-based treatments.
- Each service/treatment includes:
  - Name
  - Description/benefits
  - Duration
  - Price
  - (For device therapies) Images
- Sections: "Masaje" and "Tratamente cu dispozitive"
- Responsive layout for desktop and mobile.

#### B. Booking System

- Multi-step booking form:
  1. Contact information
  2. Service selection
  3. Date & time selection (with real-time availability)
  4. Booking summary and confirmation
- Prevent double-booking by disabling already-booked slots (real-time check).
- Store bookings in Supabase (PostgreSQL).
- Show confirmation toast and redirect after successful booking.

- Recurring bookings:
  - Users and admins can enable weekly/biweekly recurring bookings.
  - The system generates planned instances in `recurring_bookings` with `status`:
    - `true` (will occur) if slot is free
    - `false` (skipped) if slot is taken/blocked
  - Profile calendar shows skipped instances as not-green and lists them as "skipped due to already taken date".
  - Admin calendar shows only `status=true` instances and hides skipped ones.

#### C. Navigation & Layout

- Responsive navbar with desktop (inline links) and mobile (hamburger menu) modes.
- Smooth scrolling and easy navigation between sections/pages.
- All interactive elements should be touch-friendly on mobile.

#### D. Contact & Information

- Dedicated contact section with business details and a way to reach out for questions or special requests.

#### E. Visual Design

- Use images, gradients, and blur effects for a relaxing, professional look.
- Consistent use of Tailwind CSS utility classes and shadcn/ui components for styling.
- Main color: violet-400 
- Secondary color: Pink-500
- Page background theme: dark

#### F. Admin/Management

- Bookings management (`src/pages/admin/Bookings.tsx`):
  - View bookings with a calendar and per-day list.
  - Create, edit, delete bookings.
  - Create recurring bookings (preview + confirm) using Supabase Functions.
  - Calendar highlights recurring dates with `status=true` only.

- Availabilities management (`src/pages/admin/Availabilities.tsx`):
  - View and toggle per-hour availability for each day.
  - Booked slots (from `bookings` + `recurring_bookings` `status=true`) are shown as booked and cannot be toggled.
  - Create recurring availability blocks (daily/weekly/biweekly) to auto-block specific hours over 30/60/90 days with preview + confirm flow.
  - Cancel a recurring availability block to cascade-remove its generated instances.

- Audit Logs (`src/pages/admin/AuditLogs.tsx`):
  - Admin actions (booking create/update/delete, availability updates, etc.) are logged to `admin_audit_logs`.
  - Logs resolve the admin's profile (optional) and gracefully handle missing users.

### 3. Technical Requirements

- **Frontend:** React (with TypeScript), Vite, Tailwind CSS, shadcn/ui, lucide-react icons
- **State/Form:** react-hook-form for booking forms
- **Database/Backend:** Supabase (PostgreSQL) for bookings and availability
- **Routing:** react-router-dom for navigation
- **Notifications:** sonner for toast notifications
- **Date Handling:** date-fns for date logic
- **Theme:** next-themes for light/dark mode support
- **Other:** Custom hooks, utility functions, and reusable UI components

- **Supabase Edge Functions:**
  - `supabase/functions/create-recurring-bookings/` – preview/confirm planned booking instances
  - `supabase/functions/cancel-recurring-bookings/` – cancel a recurring booking series
  - `supabase/functions/create-recurring-availabilities/` – preview/confirm recurring availability blocks
  - `supabase/functions/cancel-recurring-availabilities/` – cancel a recurring availability series
  - All functions expose permissive CORS for `authorization, x-client-info, apikey, content-type` headers and run with service role.

### 4. User Stories

- As a visitor, I want to browse all available massage and device treatments, so I can choose the best option for me.
- As a user, I want to book a session by selecting a service, date, and time, so I can schedule my appointment online.
- As a user, I want to see only available time slots, so I don’t accidentally double-book.
- As a user, I want to receive a confirmation after booking, so I know my appointment is set.
- As an admin, I want to view and manage bookings and service availability.
- As an admin, I want to create recurring availability blocks (e.g., disable 13:00 for weekdays for 60 days) and cancel them safely.
- As a visitor, I want to click "Rezervă acum" on a service card and land on the booking page with that service preselected.

---

### 5. Non-Functional Requirements

- The site must be responsive and work well on both desktop and mobile devices.
- All user data must be validated before submission.
- The booking process should be fast and user-friendly.
- The design should inspire trust and relaxation.
- Mobile optimizations for calendars and grids:
  - Larger tap targets and responsive sizing for Profile/Admin calendars.
  - Availability hour grid adapts columns for small screens; booked slots are clearly indicated and disabled.

### 8. Implementation Details & Directory Map (New)

- Frontend Pages & Components
  - Booking page: `src/pages/BookingPage.tsx`
    - Reads `service` from router state or `?service=` query to preselect service.
    - Steps: Contact → Service → Date/Time → Summary → Confirm.
  - Profile page: `src/pages/ProfilePage.tsx`

    - Bookings: `src/pages/admin/Bookings.tsx` (calendar + day list, recurring booking modal)
